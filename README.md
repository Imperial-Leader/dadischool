# DadiSchool

DadiSchool is the canonical family learning operating system.

It provides the shared doctrine, standards, templates, mission flow, and reusable tooling for independent learner repositories such as:

- Mummy School
- Son School
- Personal self-school programmes
- Future subject-specific or family learning environments

## Core model

DadiSchool is the headquarters.

Learner repositories hold personal notes, assignments, essays, revision records, feedback, and progress.

Reusable improvements flow back into DadiSchool through issues and pull requests.

## Operating roles

- **Director:** defines learning objectives, approves doctrine, and governs quality.
- **ChatGPT:** discovery, curriculum architecture, study instinct, research planning, writing support, and mission design.
- **Codex:** repository infrastructure, automation, search, dashboards, validation, and reusable tooling.
- **Learner Agent:** daily study support inside the learner's own repository.
- **Learner:** completes the thinking, study, practice, and final academic work.

## Planned doctrine

- Study Instinct
- Essay Instinct
- Research Instinct
- Revision Instinct
- Exam Instinct
- Citation Instinct
- Memory Instinct
- Safeguarding and Age Adaptation
- Academic Integrity
- Agent-to-Agent Handoff Protocol

## Repository boundary

DadiSchool stores universal systems.

Learner repositories store personal learning material.

Personal notes and coursework should not be copied into DadiSchool unless they are deliberately anonymised and converted into a reusable template or doctrine improvement.
